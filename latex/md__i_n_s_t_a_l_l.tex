 \hypertarget{md__i_n_s_t_a_l_l_autotoc_md15}{}\doxysubsection{Availability}\label{md__i_n_s_t_a_l_l_autotoc_md15}
The sources for AMICI are available as
\begin{DoxyItemize}
\item Source \href{https://github.com/AMICI-dev/AMICI/tarball/master}{\texttt{ tarball}}
\item Source \href{https://github.com/AMICI-dev/AMICI/zipball/master}{\texttt{ zip}}
\item GIT repository on \href{https://github.com/AMICI-dev/AMICI}{\texttt{ github}}
\end{DoxyItemize}

A Python package is available on pypi, see below.

If AMICI was downloaded as a zip, it needs to be unpacked in a convenient directory. If AMICI was obtained via cloning of the git repository, no further unpacking is necessary.\hypertarget{md__i_n_s_t_a_l_l_autotoc_md16}{}\doxysubsubsection{Obtaining AMICI via the GIT version control system}\label{md__i_n_s_t_a_l_l_autotoc_md16}
In order to always stay up-\/to-\/date with the latest AMICI versions, simply pull it from our GIT repository and recompile it when a new release is available. For more information about GIT checkout their \href{http://git-scm.com/}{\texttt{ website}}

The GIT repository can currently be found at \href{https://github.com/AMICI-dev/AMICI}{\texttt{ https\+://github.\+com/\+AMICI-\/dev/\+AMICI}} and a direct clone is possible via \begin{DoxyVerb}git clone https://github.com/AMICI-dev/AMICI.git AMICI
\end{DoxyVerb}


\label{_python}%
 \hypertarget{md__i_n_s_t_a_l_l_autotoc_md17}{}\doxysubsection{Python}\label{md__i_n_s_t_a_l_l_autotoc_md17}
To use AMICI from python, install the module and all other requirements using pip\+: \begin{DoxyVerb}pip3 install amici
\end{DoxyVerb}


You can now import it as python module\+: \begin{DoxyVerb}import amici
\end{DoxyVerb}


For cases where this installation fails, check below for special setups and custom installations. For Python-\/\+AMICI usage see \href{https://github.com/AMICI-dev/AMICI/blob/master/documentation/PYTHON.md}{\texttt{ https\+://github.\+com/\+AMICI-\/dev/\+AMICI/blob/master/documentation/\+PYTHON.\+md}}.\hypertarget{md__i_n_s_t_a_l_l_autotoc_md18}{}\doxysubsubsection{Installation of development versions}\label{md__i_n_s_t_a_l_l_autotoc_md18}
To install development versions which have not been released to pypi yet, you can install AMICI with pip directly from Git\+Hub using\+: \begin{DoxyVerb}pip3 install -e git+https://github.com/AMICI-dev/amici.git@develop#egg=amici\&subdirectory=python/sdist
\end{DoxyVerb}


Replace {\ttfamily develop} by the branch or commit you want to install.

Note that this will probably not work on Windows which does not support symlinks by default (\href{https://stackoverflow.com/questions/5917249/git-symlinks-in-windows/49913019\#49913019}{\texttt{ https\+://stackoverflow.\+com/questions/5917249/git-\/symlinks-\/in-\/windows/49913019\#49913019}}).\hypertarget{md__i_n_s_t_a_l_l_autotoc_md19}{}\doxysubsubsection{Light installation}\label{md__i_n_s_t_a_l_l_autotoc_md19}
In case you only want to use the AMICI Python package for generating model code for use with Matlab or C++ and don\textquotesingle{}t want to bothered with any unnecessary dependencies, you can run \begin{DoxyVerb}pip3 install --install-option --no-clibs amici
\end{DoxyVerb}


Note, however, that you will not be able to compile any model into a Python extension with this installation.

NOTE\+: If you run into an error with above installation command, install all AMICI dependencies listed in \href{https://github.com/AMICI-dev/AMICI/blob/master/python/sdist/setup.py}{\texttt{ {\ttfamily setup.\+py}}} manually, and try again. (This is because {\ttfamily pip} {\ttfamily -\/-\/install-\/option}s are applied to {\itshape all} installed packages, including dependencies.)\hypertarget{md__i_n_s_t_a_l_l_autotoc_md20}{}\doxysubsubsection{Anaconda}\label{md__i_n_s_t_a_l_l_autotoc_md20}
To use an Anaconda installation of python (\href{https://www.anaconda.com/distribution/}{\texttt{ https\+://www.\+anaconda.\+com/distribution/}}, Python$>$=3.\+6), proceed as follows\+:

Since Anaconda provides own versions of some packages which might not work with amici (in particular the gcc compiler), create a minimal virtual environment via\+: \begin{DoxyVerb}conda create --name ENV_NAME pip python
\end{DoxyVerb}


Here, replace ENV\+\_\+\+NAME by some name for the environment. To activate the environment, do\+: \begin{DoxyVerb}source activate ENV_NAME
\end{DoxyVerb}


(and {\ttfamily conda deactivate} later to deactivate it again).

SWIG must be installed and available in your {\ttfamily PATH}, and a CBLAS-\/compatible BLAS must be available. You can also use conda to install the latter locally, using\+: \begin{DoxyVerb}conda install -c conda-forge openblas
\end{DoxyVerb}


To install AMICI, now do\+: \begin{DoxyVerb}pip install amici
\end{DoxyVerb}


The option {\ttfamily -\/-\/no-\/cache} may be helpful here to make sure the installation is done completely anew.

Now, you are ready to use AMICI in the virtual environment.\hypertarget{md__i_n_s_t_a_l_l_autotoc_md21}{}\doxyparagraph{Anaconda on Mac}\label{md__i_n_s_t_a_l_l_autotoc_md21}
If the above installation does not work for you, try installing AMICI via\+: \begin{DoxyVerb}CFLAGS="-stdlib=libc++" CC=clang CXX=clang pip3 install --verbose amici
\end{DoxyVerb}


This will use the {\ttfamily clang} compiler.

You will have to pass the same options when compiling any model later on. This can be done by inserting the following code before calling {\ttfamily sbml2amici}\+: \begin{DoxyVerb}import os
os.environ['CC'] = 'clang'
os.environ['CXX'] = 'clang'
os.environ['CFLAGS'] = '-stdlib=libc++'
\end{DoxyVerb}


(For further discussion see \href{https://github.com/AMICI-dev/AMICI/issues/357}{\texttt{ https\+://github.\+com/\+AMICI-\/dev/\+AMICI/issues/357}})\hypertarget{md__i_n_s_t_a_l_l_autotoc_md22}{}\doxysubsubsection{Windows using GCC (mingw)}\label{md__i_n_s_t_a_l_l_autotoc_md22}
To install AMICI on Windows using python, you can proceed as follows\+:

Some general remarks\+:


\begin{DoxyItemize}
\item Install all libraries in a path not containing white spaces, e.\+g. directly under C\+:.
\item Replace the following paths according to your installation.
\item Slashes can be preferable to backslashes for some environment variables.
\item See also \href{https://github.com/AMICI-dev/amici/issues/425}{\texttt{ \#425}} for further discussion.
\end{DoxyItemize}

Then, follow these steps\+:


\begin{DoxyItemize}
\item A python environment for Windows is required. We recommend \href{https://www.anaconda.com/distribution/}{\texttt{ Anaconda}} with python $>$=3.\+7.
\item Install \href{https://sourceforge.net/projects/mingw-w64/files/}{\texttt{ Min\+GW-\/\+W64}} (32bit will succeed to compile, but fail during linking). Min\+GW-\/\+W64 GCC-\/8.\+1.\+0 for {\ttfamily x86\+\_\+64-\/posix-\/sjlj} (\href{https://sourceforge.net/projects/mingw-w64/files/Toolchains\%20targetting\%20Win64/Personal\%20Builds/mingw-builds/8.1.0/threads-posix/sjlj/x86_64-8.1.0-release-posix-sjlj-rt_v6-rev0.7z/download}{\texttt{ direct link}} has been shown to work on Windows 7 and 10 test systems.
\item Add the following directory to {\ttfamily PATH}\+:
\begin{DoxyItemize}
\item {\ttfamily C\+:\textbackslash{}mingw-\/w64\textbackslash{}x86\+\_\+64-\/8.\+1.\+0-\/posix-\/sjlj-\/rt\+\_\+v6-\/rev0\textbackslash{}mingw64\textbackslash{}bin}
\end{DoxyItemize}
\item Make sure that this is the compiler that is found by the system (e.\+g. {\ttfamily where gcc} in a {\ttfamily cmd} should point to this installation).
\item Download CBLAS headers and libraries, e.\+g. \href{https://sourceforge.net/projects/openblas/files/v0.2.19/}{\texttt{ Open\+BLAS}}, binary distribution 0.\+2.\+19. Set the following environment variables\+:
\begin{DoxyItemize}
\item {\ttfamily BLAS\+\_\+\+CFLAGS=-\/IC\+:/\+Open\+BLAS-\/v0.2.\+19-\/Win64-\/int32/include}
\item {\ttfamily BLAS\+\_\+\+LIBS=-\/Wl,-\/Bstatic -\/LC\+:/\+Open\+BLAS-\/v0.2.\+19-\/Win64-\/int32/lib -\/lopenblas -\/Wl,-\/Bdynamic}
\end{DoxyItemize}
\item Install \href{http://www.swig.org/download.html}{\texttt{ SWIG}} (version swigwin-\/3.\+0.\+12 worked) and add the following directory to {\ttfamily PATH}\+:
\begin{DoxyItemize}
\item {\ttfamily C\+:\textbackslash{}swigwin-\/3.\+0.\+12}
\end{DoxyItemize}
\item Install AMICI using\+:

\`{}pip install --global-\/option=\char`\"{}build\+\_\+clib\char`\"{} --global-\/option=\char`\"{}-\/-\/compiler=mingw32\char`\"{} \textbackslash{} --global-\/option=\char`\"{}build\+\_\+ext\char`\"{} --global-\/option=\char`\"{}-\/-\/compiler=mingw32\char`\"{} \textbackslash{} amici --no-\/cache-\/dir --verbose\`{}
\end{DoxyItemize}

Possible sources of errors\+:


\begin{DoxyItemize}
\item On recent Windows versions, {\ttfamily anaconda3\textbackslash{}Lib\textbackslash{}distutils\textbackslash{}cygwinccompiler.\+py} fails linking {\ttfamily msvcr140.\+dll} with {\ttfamily \mbox{[}...\mbox{]} x86\+\_\+64-\/w64-\/mingw32/bin/ld.\+exe\+: cannot find -\/lmsvcr140}. This is not required for amici, so in {\ttfamily cygwinccompiler.\+py} `return \mbox{[}\textquotesingle{}msvcr140'\mbox{]}{\ttfamily can be changed to}return \mbox{[}\mbox{]}\`{}.
\item If you use a python version where \href{https://github.com/python/cpython/pull/880}{\texttt{ python/cpython\#880}} has not been fixed yet, you need to disable {\ttfamily define hypot \+\_\+hypot in anaconda3\textbackslash{}include/pyconfig.\+h} yourself.
\item {\ttfamily import amici} in python resulting in the very informative
\end{DoxyItemize}

\begin{quote}
Import\+Error\+: DLL load failed\+: The specified module could not be found. \end{quote}


means that some amici module dependencies were not found (not the AMICI module itself). \href{http://www.dependencywalker.com/}{\texttt{ Dependency\+Walker}} will show you which ones.

Support for msvc is experimental. \href{https://github.com/AMICI-dev/AMICI/blob/master/scripts/installOpenBLAS.ps1}{\texttt{ install\+Open\+BLAS.\+ps1}} and \href{https://github.com/AMICI-dev/AMICI/blob/master/scripts/compileBLAS.cmd}{\texttt{ compile\+BLAS.\+cmd}} may serve as guidance on how to install open\+BLAS using msvc.\hypertarget{md__i_n_s_t_a_l_l_autotoc_md23}{}\doxysubsubsection{Windows using MSVC (\+Visual Studio)}\label{md__i_n_s_t_a_l_l_autotoc_md23}
\hypertarget{md__i_n_s_t_a_l_l_autotoc_md24}{}\doxyparagraph{Visual Studio}\label{md__i_n_s_t_a_l_l_autotoc_md24}
We assume that Visual Studio (not to be confused with Visual Studio Code) is already installed. Using Visual Studio Installer, the following components need to be included\+:


\begin{DoxyItemize}
\item Microsoft Visual C++ (MSVC). This is part of multiple packages, including Desktop Development with C++.
\item Windows Universal C Runtime. This is an individual component and installs some DLLs that we need.
\end{DoxyItemize}\hypertarget{md__i_n_s_t_a_l_l_autotoc_md25}{}\doxyparagraph{open\+BLAS}\label{md__i_n_s_t_a_l_l_autotoc_md25}
To install open BLAS, download the following scripts from AMICI\+:

\href{https://github.com/AMICI-dev/AMICI/blob/master/scripts/installOpenBLAS.ps1}{\texttt{ https\+://github.\+com/\+AMICI-\/dev/\+AMICI/blob/master/scripts/install\+Open\+BLAS.\+ps1}} \href{https://github.com/AMICI-dev/AMICI/blob/master/scripts/compileBLAS.cmd}{\texttt{ https\+://github.\+com/\+AMICI-\/dev/\+AMICI/blob/master/scripts/compile\+BLAS.\+cmd}}

The first script needs to be called in Powershell, and it needs to call {\ttfamily compile\+BLAS.\+cmd}, so you will need to modify line 11\+: \begin{DoxyVerb}C:\Users\travis\build\AMICI\scripts\compileBLAS.cmd
\end{DoxyVerb}


so that it matches your directory structure. This will download open\+BLAS and compile it, creating \begin{DoxyVerb}C:\BLAS\lib\openblas.lib
C:\BLAS\bin\openblas.dll
\end{DoxyVerb}


You will also need to define two environment variables\+: \begin{DoxyVerb}BLAS_LIBS="/LIBPATH:C:\BLAS\lib openblas.lib"
BLAS_CFLAGS="/IC:\BLAS\OpenBLAS-v0.3.10\OpenBLAS-0.3.10"
\end{DoxyVerb}


One way to do that is to run a Power\+Shell script with the following commands\+: \begin{DoxyVerb}[System.Environment]::SetEnvironmentVariable("BLAS_LIBS", "/LIBPATH:C:\BLAS\lib openblas.lib", [System.EnvironmentVariableTarget]::User)
[System.Environment]::SetEnvironmentVariable("BLAS_LIBS", "/LIBPATH:C:\BLAS\lib openblas.lib", [System.EnvironmentVariableTarget]::Process)
[System.Environment]::SetEnvironmentVariable("BLAS_CFLAGS", "-IC:\BLAS\OpenBLAS-v0.3.10\OpenBLAS-0.3.10", [System.EnvironmentVariableTarget]::User)
[System.Environment]::SetEnvironmentVariable("BLAS_CFLAGS", "-IC:\BLAS\OpenBLAS-v0.3.10\OpenBLAS-0.3.10", [System.EnvironmentVariableTarget]::Process)
\end{DoxyVerb}


The call ending in {\ttfamily Process} sets the environment variable in the current process, and it is no longer in effect in the next process. The call ending in {\ttfamily User} is permanent, and takes effect the next time the user logs on.\hypertarget{md__i_n_s_t_a_l_l_autotoc_md26}{}\doxyparagraph{PATH}\label{md__i_n_s_t_a_l_l_autotoc_md26}
Now we need to make sure that all required DLLs are within the scope of the PATH variable. In particular, the following directories need to be included in PATH\+: \begin{DoxyVerb}C:\BLAS\bin
C:\Program Files (x86)\Windows Kits\10\Redist\ucrt\DLLs\x64
\end{DoxyVerb}


The first one is needed for {\ttfamily openblas.\+dll} and the second is needed for the Windows Universal C Runtime. If any DLLs are missing in the PATH variable, Python will return the following error\+: \begin{DoxyVerb}ImportError: DLL load failed: The specified module could not be found.
\end{DoxyVerb}


This can be tested using the \char`\"{}where\char`\"{} command. For example \begin{DoxyVerb}where openblas.dll
\end{DoxyVerb}


should return \begin{DoxyVerb}C:\BLAS\bin\openblas.dll
\end{DoxyVerb}


Almost all of the DLLs are standard Windows DLLs and should be included in either Windows or Visual Studio. But, in case it is necessary to test this, here is a list of some DLLs required by AMICI (when compiled with MSVC)\+:


\begin{DoxyItemize}
\item {\ttfamily openblas.\+dll}
\item {\ttfamily python37.\+dll}
\item {\ttfamily MSVCP140.\+dll}
\item {\ttfamily KERNEL32.\+dll}
\item {\ttfamily VCRUNTIME140\+\_\+1.\+dll}
\item {\ttfamily VCRUNTIME140.\+dll}
\item {\ttfamily api-\/ms-\/win-\/crt-\/convert-\/l1-\/1-\/0.\+dll}

\item {\ttfamily api-\/ms-\/win-\/crt-\/stdio-\/l1-\/1-\/0.\+dll}
\item {\ttfamily api-\/ms-\/win-\/crt-\/string-\/l1-\/1-\/0.\+dll}
\item {\ttfamily api-\/ms-\/win-\/crt-\/runtime-\/l1-\/1-\/0.\+dll}

{\ttfamily MSVCP140.\+dll}, {\ttfamily VCRUNTIME140.\+dll}, and {\ttfamily VCRUNTIME140\+\_\+1.\+dll} are needed by MSVC (see Visual Studio above). {\ttfamily KERNEL32.\+dll} is part of Windows and in {\ttfamily C\+:\textbackslash{}Windows\textbackslash{}System32}. {\bfseries{Algorithm 907\+: KLU}}, A Direct Sparse Solver for Circuit Simulation Problems. Timothy A. Davis, Ekanathan Palamadai Natarajan, {\itshape ACM Transactions on Mathematical Software}, Vol 37, Issue 6, 2010, pp 36\+:1 -\/ 36\+:17. \href{http://dl.acm.org/authorize?305534}{\texttt{ PDF}}

{\bfseries{Algorithm 837\+: AMD}}, an approximate minimum degree ordering algorithm, Patrick R. Amestoy, Timothy A. Davis, Iain S. Duff, {\itshape ACM Transactions on Mathematical Software}, Vol 30, Issue 3, 2004, pp 381 -\/ 388. \href{http://dl.acm.org/authorize?733169}{\texttt{ PDF}}

To import Systems Biology Markup Language (\href{http://sbml.org/}{\texttt{ SBML}}) models, AMICI relies on the Python or MATLAB SBML library.\hypertarget{md__i_n_s_t_a_l_l_autotoc_md34}{}\doxyparagraph{Math Kernel Library (\+MKL)}\label{md__i_n_s_t_a_l_l_autotoc_md34}
The python and C++ interfaces require a system installation of a {\ttfamily BLAS}. AMICI has been tested with various native and general purpose MKL implementations such as Accelerate, Intel MKL, cblas, openblas, atlas. The matlab interface uses the MATLAB MKL, which requires no separate installation.

On Ubuntu, this requirement can be satisfied with \begin{DoxyVerb}apt install libatlas-base-dev
\end{DoxyVerb}


On Fedora (32)\+: \begin{DoxyVerb}sudo dnf install blas-devel
\end{DoxyVerb}
\hypertarget{md__i_n_s_t_a_l_l_autotoc_md35}{}\doxyparagraph{C++ compiler}\label{md__i_n_s_t_a_l_l_autotoc_md35}
All AMICI installations require a C++11-\/compatible C++ compiler. AMICI has been tested with g++, mingw, clang and the Intel compiler. Visual C++ is not officially supported, but may work.\hypertarget{md__i_n_s_t_a_l_l_autotoc_md36}{}\doxyparagraph{HDF5}\label{md__i_n_s_t_a_l_l_autotoc_md36}
The python and C++ interfaces provide routines to read and write options and results in \href{https://support.hdfgroup.org/HDF5/}{\texttt{ hdf5}} format. For the python interface, the installation of hdf5 is optional, but for the C++ interace it is currently required.

HDF5 can be installed using package managers such as \href{https://brew.sh}{\texttt{ brew}} or \href{https://wiki.debian.org/Apt}{\texttt{ apt}}\+: \begin{DoxyVerb}brew install hdf5
\end{DoxyVerb}


or \begin{DoxyVerb}apt-get install libhdf5-serial-dev
\end{DoxyVerb}

The python interface requires \href{http://www.swig.org}{\texttt{ SWIG}}, which has to be installed by the user. As root user, SWIG can be installed using package managers such as \href{https://brew.sh}{\texttt{ brew}} or \href{https://wiki.debian.org/Apt}{\texttt{ apt}}\+: \begin{DoxyVerb}brew install swig
\end{DoxyVerb}


or \begin{DoxyVerb}apt-get install swig3.0
\end{DoxyVerb}


Or by non-\/root users, using {\ttfamily scripts/download\+And\+Build\+Swig.\+sh} from the AMICI repository (not included in the Py\+PI package). The binary directory has to be added to the {\ttfamily PATH} environment variable, or {\ttfamily SWIG} has to be set as described in the following section.\hypertarget{md__i_n_s_t_a_l_l_autotoc_md38}{}\doxysubparagraph{Using a non-\/default SWIG executable}\label{md__i_n_s_t_a_l_l_autotoc_md38}
We note here that some linux package managers may provide swig executables as {\ttfamily swig3.\+0}, but installation as {\ttfamily swig} is required. This can be fixed as root user using, e.\+g., symbolic links\+: \begin{DoxyVerb}mkdir -p ~/bin/ && ln -s $(which swig3.0) ~/bin/swig && export PATH=~/bin/:$PATH
\end{DoxyVerb}


Non-\/root users can set the {\ttfamily SWIG} environment variable to the full path of the desired SWIG executable. This variable has be set during AMICI package installation as well as during model compilation.\hypertarget{md__i_n_s_t_a_l_l_autotoc_md39}{}\doxysubsubsection{Matlab}\label{md__i_n_s_t_a_l_l_autotoc_md39}
The MATLAB interface requires the Mathworks Symbolic Toolbox for model generation via {\ttfamily amiwrap(...)}, but not for execution of precompiled models. Currently MATLAB R2018a or newer is not supported (see \href{https://github.com/AMICI-dev/AMICI/issues/307}{\texttt{ https\+://github.\+com/\+AMICI-\/dev/\+AMICI/issues/307}}).

The Symbolic Toolbox requirement can be circumvented by performing model import using the Python interface. The result code can then be used from Matlab.\hypertarget{md__i_n_s_t_a_l_l_autotoc_md40}{}\doxysubsubsection{Python}\label{md__i_n_s_t_a_l_l_autotoc_md40}
The python interface requires python 3.\+6 or newer and a cblas-\/compatible BLAS library to be installed. Windows installations via pip are currently not supported, but users may try to install amici using the build scripts provided for the C++ interface (these will by default automatically install the python module).

The python interface depends on some additional packages, e.\+g. {\ttfamily numpy}. They are automatically installed when installing the python package.\hypertarget{md__i_n_s_t_a_l_l_autotoc_md41}{}\doxysubsubsection{C++}\label{md__i_n_s_t_a_l_l_autotoc_md41}
